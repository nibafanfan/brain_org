# Brain Organoid Atlas — Handoff State (2026-05-20)

## Project goal

Build a brain-organoid scRNA-seq benchmark that extends HNOCA (He, Dony, Fleck et al. 2024, Nature) — which integrated 36 datasets across 26 mostly single-lineage protocols — to all 2024–2025 protocols. Control-only by construction. Eventually train scVI / scPoli for integration, then quantify organoid-to-primary correspondence against Braun 2023 primary developing brain reference.

## QC policy (uniform across the atlas)

- `n_counts ≥ 500`
- `n_genes ≥ 200`
- `pct_mito ≤ 20%`
- MT detection: namespace-aware. For Ensembl-namespace deposits, uses HNOCA canonical's 13 MT Ensembl IDs. For HGNC-namespace, uses `MT-` prefix check. **Critical:** earlier broken pipeline only did the prefix check; that bug is fixed in the current rebuild.

## Obs schema (every h5ad)

`sample_id`, `gsm`, `is_control=True`, `n_counts`, `n_genes`, `pct_mito`, `organoid_type`, `accession`, `dataset_slug`, `dataset_filter`, plus HNOCA harmonization columns: `cell_type='unknown'`, `cell_type_original`, `cell_line`, `cell_line_original`, `bio_sample`, `tech_sample`, `batch`, `individual`, `disease='healthy'`, `assay_sc`.

Smart-seq2 deposits (gse195692, gse185052, gse75140, gse124299, gse115011_pollen) additionally get `layers['counts_lengthnorm']` — currently none of these are in the atlas yet (all in the failure / pending pile).

### 2. Concatenate into single atlas file — ✅ DONE (2026-05-20)

Built `data/atlas_v4_full.h5ad` (4,255,476 × 36,842, float32 raw counts, all
control, unique obs_names). Method: `scripts/concatenate_atlas.py` projects each
deposit onto the canonical 36,842-gene set via a sparse projection matrix, writes
reindexed temp files to `data/_concat_tmp/`, then `concat_on_disk` (merge=None to
avoid an anndata 0.10.9 var-writer bug; canonical var metadata re-attached after).
File is uncompressed (~80 GB); step 3 preprocessing rewrites it. Temp dir (~76 GB)
is reproducible and safe to delete.

Original plan (for reference):
```python
# Pseudocode
import anndata as ad
from pathlib import Path
import pandas as pd

manifest = pd.read_csv('data/manifest.tsv', sep='\t')
parts = []
for _, r in manifest.iterrows():
    a = ad.read_h5ad(r['path'])
    parts.append(a)
big = ad.concat(parts, axis=0, join='outer', fill_value=0, merge='same')
big.write_h5ad('data/atlas_v4_full.h5ad', compression='gzip')
```

Memory caveat: 4.26M cells × union-of-genes may not fit in memory. The earlier atlas_v3 build hit this (see archived `processed_pre_rebuild_2026-05-17`). Strategy: subset to HNOCA canonical 36,842 genes before concat (reindex each part to canonical, then vstack). Expected output ~30–40 GB on disk.

### 3. HVG selection + normalize

Once concatenated:
- **HVG**: 4,000 highly variable genes via `sc.pp.highly_variable_genes(flavor='cell_ranger', batch_key='dataset_slug')` (NOT seurat_v3 — that needs `scikit-misc` which has a numpy ABI conflict on this Mac).
- **Normalize layers**: save raw to `layers['counts']`, then `sc.pp.normalize_total(target_sum=1e4)` + `sc.pp.log1p()` on `X`.
- Output: `data/processed/atlas_v4_preprocessed.h5ad`.

### 4. Train scVI

```python
import scvi
scvi.model.SCVI.setup_anndata(adata, layer='counts', batch_key='dataset_slug')
model = scvi.model.SCVI(adata, n_layers=2, n_latent=30, gene_likelihood='nb')
model.train(max_epochs=100)
adata.obsm['X_scvi'] = model.get_latent_representation()
```

### 6. Benchmark questions (the actual paper)

1. Do multi-lineage protocols show improved cell-type coverage vs single-lineage? (27 vs 81 deposits comparison)
2. Do added cell types (endothelium, microglia, regional neurons) transcriptomically correspond to primary brain counterparts?
3. What gaps remain even in multi-lineage systems?

## Things a fresh Claude should NOT do

- Don't rebuild the atlas from scratch — current 113 deposits + 4.26M cells is the state. Use the manifest as the entry point.
- Don't try to use union-of-all-genes for the concatenated atlas — won't fit in memory. Use HNOCA canonical 36,842-gene set as the projection target.
- Don't use `seurat_v3` HVG flavor — `scikit-misc` has numpy 2.0 ABI conflict on this Mac. Use `cell_ranger` flavor.
- Don't reuse archived `pct_mito` blindly — the v1 archive has uniformly zero `pct_mito` for Ensembl-namespace deposits (broken pipeline). The current rebuild fixes this; check that an h5ad's `pct_mito` has nonzero variance before trusting it.
- Don't include non-control cells. Everything in `data/processed/` already has `is_control=True` for every row. Don't infer control status from auto-tagged obs columns — the GSM filter already did that work.
